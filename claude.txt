1. Why the Original EZ Reader Performs So Poorly (r_TRT = 0.30-0.42)                                                                                                                                          
                                                                                                                                                                                                                
  This is the biggest red flag. EZ Reader in the literature typically gets much better fits. The problem is in compute_real_l1_l2() at line 89:                                                                 
                                                                                                                                                                                                                
  tL1 = time_familiarity_check(0, wordlen, freq, pred, eccentricity, ...)
  #                             ^ distance = 0 always

  You're passing distance=0 for every word. In the real EZ Reader, distance is the number of characters between the current fixation point and the start of the word being processed. This matters because the
  eccentricity scaling is eccentricity^(distance + (wordlen-1)/2). With distance=0, every word gets processed as if it's being fixated directly — no parafoveal penalty, no preview benefit differences. This
  kills the model's ability to distinguish between words at different retinal eccentricities.

  But that's only half the story. The simulation (ez_wrapper.py → NeuralEZReaderSim) does compute distance correctly at runtime inside visual_processing() — it uses distance = first_letter -
  self.fixation_point. So you're computing L1/L2 with distance=0, then the simulation re-applies eccentricity scaling on top:

  ecc_factor = pow(eccentricity, abs(distance + (word_len - 1) / 2.0))
  time_familiarity_check = max(0, self._neural_l1[i]) * ecc_factor

  This means the original sim gets double eccentricity on the formula L1 (once baked into the input, once applied by the sim), but with the distance=0 input, the first application is just
  eccentricity^((wordlen-1)/2). The sim then multiplies by the actual eccentricity factor. This creates a mismatch that distorts the timing.

  Also: the simulation is stochastic (saccade noise, integration failures, refixation probabilities). Even with 20 Monte Carlo runs, you get noisy estimates for 310 test words. The original EZ Reader was
  designed to be fit with thousands of simulation runs per condition.

  ---
  2. Why the Differentiable Version Performs Much Better Than the Original

  The DifferentiableEZReader with the same formula L1/L2 gets r_TRT = 0.515-0.562 vs 0.300-0.418 for the original. Same inputs, different simulation. Why?

  The differentiable version is a simpler, more predictable model. It computes TRT as:

  TRT = (1-skip_prob) * (L1*ecc + L2) + (1-skip_prob) * (saccade + attention) + integration_cost * (L1*ecc + L2)

  This is a closed-form expected value — no stochastic noise, no saccade landing errors, no cascading integration failures that cause regressions that cause more regressions. The original SimPy simulation has
   all of those, and they add variance that hurts correlation even if the mean behavior is correct.

  Think of it this way: if the true relationship is TRT ∝ f(word_properties) but your model adds random noise on top, the correlation drops even if the mean is right. The differentiable version produces the
  expected value directly, so it correlates better with the human averages (which are also expected values across participants).

  This is actually a legitimate finding for your paper: smooth approximations of cognitive models can outperform the full stochastic simulation when predicting averaged human data, precisely because averaging
   removes the stochasticity that the original model generates.

  3. Your Experimental Setup Issues

  Looking at the results, several things jump out:

  a) LSTM skip rate is wildly wrong: 0.79 predicted vs 0.45 human. Your LSTM skips almost everything. This suggests the skip head learned to output high probabilities across the board, which minimizes TRT
  loss (skip → TRT=0, which is "close" to short TRTs). The 0.3 weight on skip BCE in your loss might not be enough to counteract this.

  b) LSTM L1 is way too high: 494ms vs 86ms formula. The LSTM's L1 predictions are 5-6x the literature values. But because skip_prob is ~0.8, these get multiplied by ~0.2, producing reasonable-looking TRTs.
  The model found a degenerate solution: predict huge L1, skip almost everything, the product happens to correlate okay.

  c) BERT L2 is nearly zero: 2ms. BERT's L2 head collapsed. It's not learning a meaningful lexical access stage.

  d) Test set is tiny: 18 sentences, 310 words. With 6 text IDs in test, you have very little statistical power. The generalization gap (+0.17 for LSTM) is large but hard to interpret with so few test points.

  e) Train/test split is imbalanced: 120/13/18 sentences. The validation set (13 sentences) is too small to reliably pick the best checkpoint.

  What You're Doing                                                                                  
                                                                                                     
  You're building a Neural EZ Reader — replacing the hand-tuned cognitive parameters in the classical
   E-Z Reader eye-tracking model (Reichle et al.) with learned neural encoders (LSTM and BERT). The
  pipeline is:

  words → Neural Encoder (LSTM/BERT) → L1, L2, skip_prob → Differentiable EZ Reader → TRT, FFD, Gaze,
   Skip

  You train end-to-end on human eye-tracking data (Provo and GECO corpora) and evaluate whether the
  neural models: (a) predict reading times better than the formula baseline, and (b) still reproduce
  known psycholinguistic effects (frequency, predictability, word length).

  Does the overall idea make sense? Yes, absolutely. This is a legitimate and interesting research
  direction — bridging cognitive models with neural networks. The E-Z Reader is the right model to
  "neuralize" since it's the dominant serial-attention model in reading research. The approach of
  keeping the cognitive architecture (L1→L2→saccade pipeline) but learning the input mapping is
  well-motivated.

  ---
  What You Got Right

  1. The differentiable approximation is clever. Replacing SimPy's discrete-event queue with smooth
  tensor ops so you can backprop through the cognitive model is the key insight. It works — Diff EZ
  alone (r=0.56) already beats original EZ (r=0.42).
  2. Training on per-participant data, evaluating on aggregated data. This is the correct strategy —
  you get ~12K training observations instead of 151, and aggregated evaluation gives you stable
  targets.
  3. Splitting by text_id. Prevents data leakage since sentences within the same passage share
  context.
  4. Cross-corpus evaluation (GECO→Provo). This is the strongest evidence. BERT trained on GECO
  achieves r=0.618 on Provo — a corpus it never saw. That's real generalization.
  5. Psycholinguistic effects analysis. Testing whether your models reproduce frequency,
  predictability, and length effects is exactly what reviewers would ask for. All models pass.

  ---
  Crucial Design Problems

  1. The LSTM skip head has found a degenerate solution

  Your LSTM predicts skip_rate = 0.79 vs human 0.45. Look at the per-word predictions — it skips
  almost everything with probability 0.65-0.90. It's compensating by predicting extremely high L1
  values (~494ms vs formula's 87ms). The model has learned: "predict high skip to minimize skip BCE
  loss on frequently-skipped words, then crank up L1 to still get reasonable TRT for the few times it
   does fixate."

  This is a loss function incentive problem. TRT = (1-skip) * (gaze + overhead), so the model can get
   low TRT loss with many different skip/L1 combinations. Your loss weights (0.3 TRT, 0.2 FFD, 0.2
  Gaze, 0.3 Skip) don't prevent this degeneracy.

  2. BERT's L2 head has collapsed (Provo)

  BERT predicts L2 ≈ 2ms (Provo) — it's effectively dead. The model is routing all its signal through
   L1 and ignoring the L2 stage entirely. On GECO it's better (L2 ≈ 22ms), but this means the
  "two-stage" architecture (familiarity check → lexical access) that's supposed to be the whole
  theoretical contribution isn't actually learned on the smaller dataset.

  Root cause: The BERT training loss has no gaze duration component — only TRT, FFD, and Skip.
  Without gaze loss (where Gaze = L1 + L2), there's no gradient pressure forcing L2 to be non-zero.
  The LSTM version has gaze loss and doesn't collapse as badly.

  3. Your DifferentiableEZReader oversimplifies regressions

  The original EZ Reader has explicit integration failure → regression → re-reading cycles. Your
  smooth version replaces this with:
  integration_penalty = softplus(integration_cost) * (L1 + L2)
  This is just a linear scaling of processing time. Real regressions involve returning to earlier
  words, re-reading them, and then coming back. This matters because TRT includes re-reading time
  from regressions initiated by later words, not just the current word's own difficulty. Your model
  can't capture this inter-word dependency in TRT.

  4. FFD predictions are systematically too low

  Across all models and both corpora, predicted FFD is ~60-130ms vs human ~210ms. This is because
  your first_fixation = L1_scaled in the differentiable EZ Reader, but real FFD includes not just L1
  but also partial L2 processing and a saccade latency component. The model underestimates FFD
  because the theory mapping is too simple.

  5. The eccentricity approximation drops a key mechanism

  Original EZ Reader scales L1 by eccentricity^(distance_to_word) where distance is the actual
  fixation-word offset in character positions (depends on where the eye is right now). Your
  approximation:
  ecc_scale = 1.0 + eccentricity * (word_lengths - 4.0).clamp(min=0)
  This only uses word length, not position relative to current fixation. It's missing the parafoveal
  preview benefit — that words closer to fixation get processed faster, which is one of the core EZ
  Reader mechanisms.

  6. The effect magnitudes are wildly over-predicted

  Your models get the effect directions right (pass/fail), but the magnitudes are often 200-1800% of
  human effect sizes. For example, the frequency effect on FFD is 1297% of human for Original EZ.
  This means your models are massively over-sensitive to frequency/length. This could be a problem
  for a paper — reviewers will notice.

  ---
  Could It Be Made Better?

  High-impact fixes:

  1. Add gaze loss to the BERT training (trivially easy — just copy the LSTM loss function). This
  alone should fix the L2 collapse.
  2. Constrain the skip-L1 tradeoff. Options:
    - Add a KL divergence penalty pushing mean skip probability toward the corpus mean (~0.45)
    - Add L1/L2 range regularization (e.g., penalize L1 > 200ms)
    - Use a harder loss weighting: increase skip_loss weight and add an explicit penalty for
  mean(skip_prob) deviating from the observed base rate
  3. Fix the FFD formula. Real FFD ≈ L1 + partial_L2 + motor_latency. At minimum, add a learnable FFD
   offset parameter:
  first_fixation = L1_scaled + self.ffd_offset  # where ffd_offset ~ 100ms

  Medium-impact improvements:

  4. Model regressions explicitly. Instead of integration_penalty = cost * (L1+L2), model the
  probability of regression as sigmoid(regression_sharpness * (L2 - threshold)) and the regression
  cost as going back to re-read the previous word. This would make TRT predictions more realistic.
  5. Use the actual GECO predictability effect analysis correctly. Your GECO predictability bins are
  broken — Zero pred: N=0 means you have no zero-predictability words, so all the predictability
  effect cells show N/A. This is because GPT-2 surprisal never gives exactly 0 predictability. You
  should use tertile splits instead of zero/low/high.
  6. Larger/better data split for Provo. 13 validation sentences (249 words) is too few for reliable
  model selection. Consider k-fold cross-validation, or at least a 70/15/15 split.

  Lower priority but good for a paper:

  7. Report confidence intervals. Run with 5 random seeds and report mean ± std for all metrics. A
  single seed on 310 test words is noisy.
  8. Ablation study. Show what happens when you remove the differentiable EZ Reader and just predict
  TRT directly from LSTM/BERT. This isolates whether the cognitive architecture helps or if it's just
   the neural encoder doing the work.

  ---
  Bottom Line

  The core idea is sound and publishable. The results are promising — especially the cross-corpus
  generalization. But the execution has a few real problems (degenerate skip, collapsed L2,
  oversimplified regressions) that weaken the theoretical story. The good news is most of these are
  fixable with relatively small changes to the loss function and the differentiable EZ Reader
  formulas. The biggest conceptual weakness is that the "differentiable EZ Reader" has diverged far
  enough from the actual EZ Reader architecture that you'd need to carefully argue why it's still
  theoretically meaningful rather than just being a fancy regression model with cognitive-sounding
  variable names.

  3. Testing the Architecture Itself

  Here's another angle nobody has done. You can ablate the cognitive architecture:

  - What happens if you remove the two-stage structure and just have one processing time? Does performance drop?
  - What if you remove the regression mechanism? Which effects break?
  - What if you remove eccentricity scaling?

  Each ablation tests whether that component of E-Z Reader is actually necessary. If you remove L2 and everything still works fine —
   that's evidence against the two-stage theory. If removing regressions kills the TRT predictions but not FFD — that tells you
  something specific about what regressions contribute.

  This is something only your differentiable version can do. You can't ablate the original E-Z Reader cleanly because you can't
  retrain it.

  **************************************************                                                                                                                                  
  The original formula says:                                                                                                        
  L1 = α1 - α2 × ln(frequency) - α3 × predictability                                                                                
                                                                                                                                    
  This means the formula thinks two words with the same frequency and predictability are equally hard to recognize. That's almost
  certainly wrong. Your neural network isn't limited this way — it can pick up on anything in the text. The question is: does it?

  How You'd Test This

  Step 1: Find words where the neural model and the formula disagree.

  Take your test set. Run both models. Find words where:
  - The formula predicts low L1 (easy) but the neural net predicts high L1 (hard)
  - Or vice versa

  These disagreement words are where the neural network has learned something the formula can't express.

  Step 2: Ask — what's special about those words?

  Analyze the disagreement words. Group them by properties the formula ignores:

  ┌─────────────────────────┬────────────────────────────────────────────────────────────┬────────────────────┬─────────────────┐

  ├─────────────────────────┼────────────────────────────────────────────────────────────┼────────────────────┼─────────────────┤

  ├─────────────────────────┼────────────────────────────────────────────────────────────┼────────────────────┼─────────────────┤
  │ Syntactic role          │ "run" as noun vs "run" as verb                             │ No                 │ Yes             │
  ├─────────────────────────┼────────────────────────────────────────────────────────────┼────────────────────┼─────────────────┤
  │ Semantic fit with       │ "The doctor baked a cake" — "doctor" is common but         │ No                 │ Yes             │
  │ context                 │ surprising here                                            │                    │                 │
  ├─────────────────────────┼────────────────────────────────────────────────────────────┼────────────────────┼─────────────────┤

  ├─────────────────────────┼────────────────────────────────────────────────────────────┼────────────────────┼─────────────────┤
  │ Number of syllables     │ "cat" vs "apt" (same length, different syllable structure) │ No                 │ Maybe           │

  If you find that the neural network consistently assigns higher L1 to morphologically complex words even when frequency is
  controlled — you've discovered that the E-Z Reader formula is missing morphological complexity as a predictor of recognition time.
   That's a publishable finding about reading, not about neural networks.

  Step 3: Controlled comparisons.

  Pick minimal pairs from your data:

  Same frequency, same predictability, different neural L1:
    "bright" (L1 = 62ms) vs "knight" (L1 = 94ms)
    → Why? Maybe orthographic irregularity ("kn-" silent letter)

  Same word, different contexts, different neural L1:
    "bank" in "He sat by the river bank" (L1 = 78ms)
    "bank" in "She went to the bank" (L1 = 65ms)
    → The formula gives identical L1. The neural net doesn't.
    → Is the difference real? Check against human data.

  Run a regression analysis:
  residual_L1 = neural_L1 - formula_L1

  Then: residual_L1 ~ morphological_complexity + syntactic_surprisal +
                       orthographic_regularity + semantic_similarity + ...

  Whichever predictors explain the residual are what the neural network learned beyond the formula. You're essentially using the
  neural network as a discovery tool to find missing variables in the cognitive model.

  Why This Matters

  If you find that, say, morphological complexity explains 15% of the residual variance — you'd propose:
  L1_improved = α1 - α2 × ln(freq) - α3 × pred - α4 × morphological_complexity

  You've used the neural network to improve the original theory. The neural net found a gap, you identified what fills it, and now
  the formula-based model can be updated. The neural network was a telescope — you looked through it, found something, and now you
  can describe what you found without needing the telescope.

  3. Ablation Studies — Testing the Architecture Itself

  E-Z Reader makes several strong theoretical claims. Each claim corresponds to a component in your differentiable model. You can
  remove each component, retrain, and see what breaks.

  The Claims and Their Ablations

  Claim 1: Reading has TWO stages (L1 then L2)

  Ablation: merge them into one.
  # Current: two separate heads
  L1 = self.l1_head(projected)  # recognition
  L2 = self.l2_head(projected)  # comprehension

  # Ablation: single processing time
  L_total = self.single_head(projected)
  L1 = 0.4 * L_total  # fixed split
  L2 = 0.6 * L_total

  If performance barely changes → maybe one stage is enough, the two-stage distinction isn't necessary.
  If performance drops significantly → the model needs independent control of recognition and comprehension, validating the
  two-stage theory.

  Claim 2: Eccentricity matters (words far from fixation are harder to process)

  Ablation: remove it.
  # Current
  ecc_scale = 1.0 + self.eccentricity * (word_lengths - 4.0).clamp(min=0)
  L1_scaled = L1 * ecc_scale

  # Ablation
  L1_scaled = L1  # no eccentricity scaling

  If word length effects disappear → eccentricity scaling was doing the heavy lifting for length effects.
  If word length effects persist → the neural network learned length effects on its own, eccentricity scaling is redundant.

  Claim 3: Difficult words cause regressions to previous words

  Ablation: remove the regression mechanism entirely.
  # Current
  regression_prob = sigmoid(sharpness * (L2 - threshold))
  regression_penalty = regression_prob * cost * prev_gaze

  # Ablation
  regression_penalty = 0  # no regressions

  This one is particularly interesting. It should hurt TRT (which includes re-reading) but NOT FFD (first fixation doesn't involve
  regressions). If removing regressions hurts FFD too — something is wrong with the theory, regressions shouldn't affect first
  fixation.

  Claim 4: Skip probability is a separate decision from processing time

  Ablation: derive skip from L1 instead of having a separate head.
  # Current: independent skip head
  skip_prob = self.skip_head(projected)

  # Ablation: skip derived from L1
  skip_prob = sigmoid(-L1)  # easy words (low L1) get skipped

  If performance is similar → skip is just a function of recognition speed, no separate mechanism needed.
  If performance drops → skip decisions involve information beyond word processing difficulty.

  Claim 5: L2 contributes partially to first fixation duration

  Ablation: FFD from L1 only.

  L1 = α1 - α2 * ln(freq) - α3 * pred                                                                                               
  L2 = δ * (α1 - α2 * ln(freq) - α3 * pred)                                                                                         
                                                                                                                                    
  L2 is just L1 multiplied by δ (delta). And delta = 0.34 in the default parameters.                                                
                  
  So in the original E-Z Reader: L2 ≈ 0.34 × L1. L2 is smaller than L1, not larger.

  For a concrete example — a medium-frequency word (freq=100,000, pred=0.2):
  L1 = 104 - 3.4 * ln(100000) - 39 * 0.2 = 104 - 39.1 - 7.8 = 57.1 ms
  L2 = 0.34 * 57.1 = 19.4 ms

  So in the original theory, L1 is the longer stage and L2 is about a third of L1. The naming is a bit counterintuitive —
  "familiarity check" (L1) takes longer than "lexical access" (L2) — but that's how Reichle et al. parametrized it.

  right; it just needs to stop decaying.

● OB1-Reader: How it works                                                                           
                                                                                                     
  OB1-Reader (Snell et al., 2018, Psychological Review) is a parallel processing model — the opposite
   camp from E-Z Reader (which is serial). Key architecture:                                         
                                                                                                     
  1. Visual input: Text around fixation, with acuity falloff (Gaussian attention window)             
  2. Open bigram layer: Letters activate position-independent letter pairs (e.g., "reading" → "re",  
  "ra", "rd", "ea"...) — borrowed from isolated word recognition research                            
  3. Word node layer: Bigrams activate word candidates via interactive activation (excitation,       
  lateral inhibition, decay). Frequency and predictability modulate activation.                      
  4. Recognition threshold: Word is "recognized" when activation exceeds a threshold based on        
  frequency and word length                                                                          
  5. Saccade system: Stochastic saccade planning in 25ms time steps, adaptive attentional window     
  (widens after easy recognition, narrows after difficulty)

  1. The skip mechanism is the biggest problem                                                                                      
                  
  DiffEZR does:
  TRT = (1 - skip_prob) * (gaze + overhead + regression)

  This is a soft expectation. A word with skip_prob=0.7 still contributes 30% of its reading time to TRT. The model learns L1/L2

  The simulation does a hard binary draw. Either you fixate fully or you get TRT=0. No in-between.

  long does L1 take? How long does L2 take?"

  Right now the model is learning E[TRT]. You want it to learn E[TRT | fixated].

  2. FFD formula is too disconnected from simulation dynamics

  DiffEZR: FFD = L1_scaled + softplus(l2_contribution) * L2

  Simulation: FFD is the actual time the eyes sit on the word during first fixation. That's roughly L1 + however much of L2 elapses
  before the saccade to the next word fires. The saccade is triggered by L1 completion (which starts M1 programming), then M1+M2
  execute in parallel with L2. So FFD ≈ L1 + min(L2, M1+M2).

  Fix: Make the FFD formula closer to the simulation:
  FFD = L1 + min(L2, saccade_time)
  or softened:
  FFD = L1 + L2 * sigmoid(saccade_time - L2)
  This captures the key dynamic — L2 only contributes to FFD if it finishes before the saccade lands on the next word.

  3. No sequential dependencies

  DiffEZR processes all words in parallel. The simulation is sequential — skipping word N changes your fixation position, which
  changes eccentricity for word N+1, which changes everything downstream.

  Fix: Add a lightweight left-to-right pass after computing L1/L2/skip. Track a soft "fixation position" state. Each word's
  eccentricity depends on whether previous words were skipped. Still differentiable, but captures the cascade. This doesn't need to
  be as complex as the full simulation — even a simple recurrence that adjusts eccentricity based on accumulated skip probability
  would help.

  4. Gaze duration needs to drive L2, not the other way around

  Currently gaze loss weight is 0.4 (high) to prevent L2 collapse. But L2 is still collapsing to ~24ms while L1 is ~103ms. The L2/L1
   ratio is 0.23 — that's inverted from cognitive plausibility (L2 should be longer than L1).

  Fix: Gaze = L1 + L2, so if you train Gaze well and FFD well (which is mostly L1), then L2 = Gaze - FFD naturally gets the right
  range. Restructure the loss to supervise Gaze and FFD directly as the primary signals, and derive the L1/L2 constraints from their
   relationship rather than regularizing them independently.

   1. Motor latency floor (fixes -82ms FFD bias)                                                                                     
   
  The problem in detail. Look at the DiffEZReader formula for FFD:                                                                  
                  
  # diff_ezreader.py:85                                                                                                             
  first_fixation = L1_scaled + softplus(l2_contribution) * L2_scaled                                                                
                                                                                                                                    
  This says FFD = word processing time only. But human FFD has a physiological floor of ~150ms even for the easiest words — you need
   time for saccade planning (M1), saccade execution (M2), and retinal processing before any cognitive processing begins. Your model
   can't output FFD < L1, so L1 is forced to encode both "motor baseline" and "cognitive difficulty" in one number. That's why mean 
  predicted FFD is 82ms below human.

  Interestingly, TRT already has an overhead term:                                                                                  
  # diff_ezreader.py:103-104
  overhead = self.saccade_time + self.attention_shift  # ~175ms                                                                     
  total_reading_time = fixate_prob * (gaze_duration + overhead + regression_penalty)                                                
                                                                                                                                    
  But FFD and Gaze have no such term. That's the asymmetry causing the bias.                                                        
                                                                                                                                    
  The fix. Add a motor latency to FFD/Gaze:                                                                                         
  self.motor_latency = nn.Parameter(torch.tensor(100.0))                                                                            
  first_fixation = self.motor_latency + L1_scaled + softplus(self.l2_contribution) * L2_scaled                                      
  gaze_duration = self.motor_latency + L1_scaled + L2_scaled                                                                        
                                                                                                                                    
  Expected difference:                                                                                                              
  - FFD bias goes from -82ms → ~0ms                                                                                                 
  - L1 no longer needs to encode motor planning — it can focus purely on cognitive word difficulty, making it more interpretable    
  - L1 values would shift down by ~80-100ms, with the motor_latency absorbing the constant                                          
  - Accuracy might improve slightly because the model has a better-structured output space                                          
                                                                                                                                    
  ---                                                                                                                               
  2. L1 ↔ surprisal anchor                                                                                                          
                                                                                                                                    
  The problem. L1 is supposed to be "familiarity check" — how quickly the reader recognizes a word. In information theory, word     
  surprisal (negative log probability in context) is the best-known measure of word processing difficulty. If L1 doesn't correlate  
  with surprisal, it's not really capturing what E-Z Reader theory says it should.
                                                                                                                                    
  Right now there's no constraint, so L1 could be anything — the optimizer just finds two numbers that minimize reading time loss   
  when combined through the EZR formulas.
                                                                                                                                    
  The fix. LLaMA is a causal LM, so you can compute surprisal as a free byproduct. During forward pass:                             
   
  # Get LLaMA output WITH logits (need lm_head)                                                                                     
  outputs = self.llama(input_ids, attention_mask)                                                                                   
  hidden = outputs.last_hidden_state                                                                                                
                                                                                                                                    
  # Compute surprisal from the LM head                                                                                              
  with torch.no_grad():                                                                                                             
      logits = self.lm_head(hidden)  # need to keep lm_head                                                                         
      log_probs = F.log_softmax(logits, dim=-1)                                                                                     
      # surprisal of each token = -log_prob of next token                                                                           
      surprisal = -log_probs.gather(2, input_ids[:, 1:].unsqueeze(-1)).squeeze(-1)                                                  
      word_surprisal = pool_to_words(surprisal)  # aggregate per word                                                               
                                                                                                                                    
  # Soft constraint: L1 should correlate with surprisal                                                                             
  l1_centered = L1 - L1.mean()                                                                                                      
  surp_centered = word_surprisal - word_surprisal.mean()                                                                            
  corr = (l1_centered * surp_centered).sum() / (L1.std() * word_surprisal.std() + 1e-8)
  surprisal_loss = LAMBDA_SURP * (1.0 - corr)  # maximize correlation                                                               
                                                                                                                                    
  Expected difference:                                                                                                              
  - L1 becomes interpretable as "information-theoretic word difficulty" — you can make a plot showing L1 vs surprisal with r=0.7+   
  - L2 is forced to capture something different (since L1 is anchored). It might end up capturing morphological complexity, word    
  length effects, or integration difficulty                                                                                     
  - The paper can then say: "L1 maps to surprisal (early word recognition), L2 captures additional processing not explained by      
  surprisal (deeper lexical access)"                                                                                          
  - Accuracy impact: probably neutral. If L1 ≈ surprisal is already roughly what the unconstrained model learns, the anchor just    
  confirms it. If not, slight accuracy loss but huge interpretability gain                                                      
  - This is the strongest claim you can make in the paper — it connects E-Z Reader theory to information theory                     
                  
  ---                                                                                                                               
  3. Layer separation (L1 from middle layers, L2 from top layers)
                                                                                                                                    
  The problem. Both L1 and L2 read from the same representation:
  projected = self.projection(word_repr)  # same 256-dim vector                                                                     
  L1 = self.l1_head(projected)  # reads from projected         
  L2 = self.l2_head(projected)  # reads from SAME projected                                                                         
                                                                                                                                    
  They share all information. There's nothing preventing L1 and L2 from capturing the same thing, or from splitting arbitrarily.    
                                                                                                                                    
  The fix. LLaMA layers form a hierarchy: lower layers capture surface/orthographic features, upper layers capture                  
  semantic/contextual features. Map this onto E-Z Reader's two stages:                                                              
                                                                                                                                    
  outputs = self.llama(input_ids, attention_mask, output_hidden_states=True)
  all_hidden = outputs.hidden_states  # tuple of (B, seq, 2048) per layer                                                           
                                                                                                                                    
  # L1: mid-level features (orthographic/phonological → familiarity)                                                                
  l1_hidden = self._pool_subwords(all_hidden[n_layers // 2], word_maps, ...)                                                        
  L1 = self.l1_head(self.proj_l1(l1_hidden)) * self.l1_scale                                                                        
                                                                                                                                    
  # L2: top-level features (semantic/contextual → lexical access)                                                                   
  l2_hidden = self._pool_subwords(all_hidden[-1], word_maps, ...)                                                                   
  L2 = self.l2_head(self.proj_l2(l2_hidden)) * self.l2_scale                                                                        
                                                                                                                                    
  # Skip: top-level (needs full context to predict skipping)                                                                        
  skip = self.skip_head(self.proj_skip(l2_hidden))                                                                                  
                                                                                                                                    
  Expected difference:                                                                                                              
  - L1 and L2 are naturally decorrelated because they read from different representations                                           
  - L1 from middle layers would be more sensitive to word form (length, orthographic regularity) — consistent with "familiarity     
  check"                                                                                                                       
  - L2 from top layers would be more sensitive to meaning in context — consistent with "lexical access"                             
  - You could ablate which layer feeds L1 and show a sweet spot (too low = no linguistic info, too high = redundant with L2)
  - Potential accuracy improvement if the separation matches how LLaMA actually organizes information                               
  - Risk: if the assumption is wrong (LLaMA doesn't neatly separate surface vs semantic by layer depth), this could hurt            
                                                                                                                                    
  ---                                                                                                                               
  The convergence problem — this is critical

  The core issue: Your LLM (the part that actually reads words) is learning 50x slower than the output layer, and the output layer  
  can find a decent solution by itself by just adjusting scales. The LLM never gets a chance to contribute fine-grained features.

  2. Narrow the LR gap. The 50x gap between LLM and EZR is too large:                                                               
  lm_lr = 5e-5        # increase from 2e-5                                                                                          
  head_lr = 2e-4      # decrease from 5e-4
  ezr_lr = 2e-4       # decrease from 1e-3 (same as heads)                                                                          
                                                                                                                                    
  This forces the model to rely more on LLaMA features rather than scale hacking.                                                   
                                                                                                                                    
  3. Freeze EZR parameters for the first N epochs. Let the LLM learn word discrimination first, then unfreeze EZR to adapt:         
  # First 5 epochs: freeze EZR, let LLM+heads learn word representations                                                            
  if epoch <= 5:                                                                                                                    
      for p in model.ezreader.parameters():                                                                                         
          p.requires_grad = False          
  else:                                                                                                                             
      for p in model.ezreader.parameters():                                                                                         
          p.requires_grad = True                                                                                                    
                                                                                                                                    
  4. Smaller effective batch size. You're at 8×8=64 effective. Ohio State uses 4. Larger batches smooth gradients but reduce the    
  number of weight updates per epoch. Try:                                                                                          
  batch_size = 4                                                                                                                    
  accumulation_steps = 4  # effective = 16                                                                                          
                                          
  5. Replace ReduceLROnPlateau with cosine or linear schedule. Plateau scheduler is dangerous because it can kill learning rate too 
  early. Cosine schedule guarantees the LLM keeps learning throughout training:                                                     
  scheduler = torch.optim.lr_scheduler.CosineAnnealingLR(                                                                           
      optimizer, T_max=num_epochs, eta_min=1e-6                                                                                     
  )

● The key thing to understand first: there is no equation for FFD, Gaze, TRT, or Skip in E-Z Reader. They are
  emergent from a discrete-event simulation. The model outputs a sequence of (word, fixation_onset, fixation_offset)
  tuples, and the reading time measures are read off that trace.

  Here's how that works.

  ---
  E-Z Reader is a simulator, not a formula

  The model runs a timeline of stages. The stages themselves have durations given by formulas (L1, L2, M1, M2, V, A,
  I — see my previous answer). But FFD/Gaze/TRT/Skip come from watching where the simulated eye is over time.

  The state machine looks roughly like this:

  Eye lands on word N at time t0
      ↓ V = 50 ms       visual encoding delay
      ↓ L1 on N         familiarity check begins
          When L1 completes → start M1 (labile saccade program to N+1)
      ↓ L2 on N         lexical access begins (in parallel with M1)
          When L2 completes → attention shifts to N+1 (A = 50 ms)
          On word N, integration stage t(I) = 25 ms begins
          With prob pF: integration fails → cancel saccades, regress
      ↓ M1 → M2 → S     saccade executes
  Eye leaves N, lands on N+1 at time t1

  The fixation on N starts at t0 and ends at t1. That's it. Everything else is bookkeeping on multiple fixations.

  ---
  FFD (First Fixation Duration)

  Definition: The duration of the first time the eye fixates on word N during first-pass reading.

  How it emerges: When the eye lands on N (at t0), the clock starts. When a saccade executes moving the eye off N
  (whether to N+1 or to a refixation within N or to a regression), the clock stops. The difference is one fixation
  duration. FFD is the duration of the first such fixation on N.

  Timing: A typical first fixation ends when M1+M2 completes for the saccade off the word. Since M1 starts when L1
  completes (or when integration failure occurs, whichever comes first), FFD is approximately:

  FFD ≈ t(L1)' + M1 + M2
      = [eccentric L1] + 125 + 25 ms

  for a single-fixation word with no regression. But this isn't an equation used by the model — it's an observed
  consequence of the simulation when you trace it. If L2 finishes before M1 starts (fast word), M1 can be cancelled
  and FFD can be shorter. If integration fails, FFD is extended by the integration stage.

  Conditional on fixation: FFD is only defined when the word is actually fixated. Skipped words have no FFD.

  ---
  Gaze Duration

  Definition: The sum of all first-pass fixations on word N, before the eye leaves N for a later word. If N is
  fixated once, Gaze = FFD. If N is refixated, Gaze = FFD + (subsequent fixation durations on N during the first
  pass).

  How it emerges: After a fixation on N ends, the simulation checks whether to refixate. Refixation is triggered by
  saccadic landing error:

  P(refixate) = λ · | landing_position − word_center |
  λ = 0.05 (E-Z Reader 10) or 0.16 (E-Z Reader 7)

  If the programmed saccade landed far from the optimal viewing position (word center), a corrective refixation is
  probabilistically triggered before the eye moves on. The simulation generates another fixation on N, which runs its
   own M1/M2 cycle, and then the eye either refixates again or moves on.

  So:

  Gaze(N) = FFD(N) + Σ (duration of each refixation on N)

  where the sum is over zero, one, or more refixations. Gaze = FFD when there are no refixations; Gaze > FFD when
  there are.

  Crucially: refixation does NOT depend on L1 or L2 in the original model. It depends on where the saccade landed. A
  long word → more likely to land off-center → more refixations → Gaze >> FFD. A short word → tends to land on center
   → no refixations → Gaze = FFD.

  This is why length effects on Gaze are much stronger than on FFD in human data — and why E-Z Reader reproduces that
   pattern.

  ---
  TRT (Total Reading Time / Total Fixation Time)

  Definition: The sum of all fixations on word N, including re-reading fixations that happen after the eye has
  already moved past N. So TRT = Gaze + (any refixations on N triggered by regressions from later words).

  How it emerges: Regressions are triggered by the integration stage. After L2 on word M completes, there is an
  integration stage t(I) = 25 ms, and with probability pF (~0.01) the integration fails. On failure:

  1. All pending saccades are cancelled
  2. A regression is programmed back to the "problem word"
  3. With probability pN the regression targets M itself; otherwise some other word
  4. The simulation now fixates the problem word again, runs L1/L2/integration again
  5. When that succeeds, reading continues forward

  If word N was the target of such a regression (from some later word M), N gets another fixation. That fixation's
  duration adds to TRT(N) but not to Gaze(N), because Gaze only counts first-pass fixations.

  TRT(N) = Gaze(N) + Σ (duration of re-reading fixations on N)

  TRT = Gaze for words that are never regressed to. TRT > Gaze for words that get re-read.

  ---
  Skip

  Definition: The probability that word N receives no fixation at all during first-pass reading.

  How it emerges: Skipping is implemented as a gate inside L1:

  if random_draw < cloze(N):   t(L1 on N) = 0
  else:                         t(L1 on N) = α1 − α2·ln(freq) − α3·pred

  This happens during parafoveal preview — i.e., when the eye is still on word N−1 and attention is allocated to N.
  If the draw succeeds (cloze > draw), L1 on N takes zero time, which means:

  - L1 on N completes "instantly" during preview
  - M1 for the saccade off N−1 has already been programmed to target N
  - But since L1 on N is already done, attention can shift past N
  - The next M1 programs a saccade to N+1 instead, cancelling the saccade to N
  - The eye jumps from N−1 to N+1, and N is never fixated

  So P(skip N) ≈ cloze(N) at the base rate, modulated by timing constraints (whether L1 on N finishes before the
  saccade to N commits to non-labile M2).

  In practice the observed skip rate is lower than cloze because the L1=0 gate has to fire and the saccade
  cancellation has to succeed before M2 locks in. The simulation handles all of this mechanically.

  Important: In E-Z Reader, skip probability has no dedicated formula. It's P(skip) = P(L1 = 0 AND saccade-to-N is
  still cancellable when L1 finishes). The whole thing emerges from the race between L1 completion and the M1→M2
  pipeline.

  ---
  Why this matters for your implementation

  model_llama_faithful_sh.py tries to turn all four of these emergent quantities into direct functions of L1, L2, and
   a skip head. But in the original:

  ┌─────────┬──────────────────────────────────────────────────────┬─────────────────────────────────────────────┐
  │ Measure │                 Original E-Z Reader                  │                  Your code                  │
  ├─────────┼──────────────────────────────────────────────────────┼─────────────────────────────────────────────┤
  │ FFD     │ Emerges from L1_onset → M1 → M2 timing               │ Hardcoded: FFD = L1                         │
  ├─────────┼──────────────────────────────────────────────────────┼─────────────────────────────────────────────┤
  │ Gaze    │ Emerges from FFD + refixation loop driven by landing │ Hardcoded: Gaze = L1 + L2                   │
  │         │  error                                               │                                             │
  ├─────────┼──────────────────────────────────────────────────────┼─────────────────────────────────────────────┤
  │ TRT     │ Emerges from Gaze + regression re-reading triggered  │ Hardcoded: TRT = Gaze + sigmoid(L2) ·       │
  │         │ by pF                                                │ prev_gaze                                   │
  ├─────────┼──────────────────────────────────────────────────────┼─────────────────────────────────────────────┤
  │ Skip    │ Emerges from L1 = 0 race against saccade commitment  │ Separate head: sigmoid(LLaMA features)      │
  └─────────┴──────────────────────────────────────────────────────┴─────────────────────────────────────────────┘

  The fundamental mismatch is this: in Reichle, Gaze > FFD because of refixations driven by saccadic landing error,
  and TRT > Gaze because of regressions driven by integration failure. Both mechanisms are stochastic events driven
  by factors independent of L1/L2 themselves — landing error depends on saccade noise, integration failure depends on
   sentence-level properties.

  Your code has neither mechanism. You made FFD, Gaze, and TRT all direct linear/near-linear functions of L1 (via L2
  = δ·L1 and a tiny regression term). That's why FFD and Gaze in your model are perfectly correlated: you have no
  refixation stage to break them apart.

  The minimal thing to add, if you want faithfulness on this axis specifically:

  1. Refixation gate using word length as a proxy for saccadic landing error (longer word → more likely to land
  off-center → more refixation):
  P_refix = sigmoid( λ · (wordlen − c) )
  Gaze = FFD + P_refix · (second_fixation_duration)
  1. where second_fixation_duration can be its own learned quantity or can reuse L2.
  2. Regression gate with a constant pF, not a sigmoid on L2:
  P_regress = sigmoid(pF_logit)   # learnable scalar, init at logit(0.01)
  TRT = Gaze + P_regress · (rereading_cost)
  3. Skip as a race condition, not a separate head. The cleanest differentiable version:
  # skip happens when L1 would be zeroed before the saccade commits
  P_skip = cloze_from_LLM · sigmoid( k · (M1_remaining − L1) )
  3. which is a soft relaxation of "L1 finishes before M1 enters non-labile phase".

  With these three additions, your four outputs have actually-distinct origins — refixation drives Gaze−FFD,
  regression drives TRT−Gaze, the L1/M1 race drives Skip — mirroring how Reichle's simulator produces them.
